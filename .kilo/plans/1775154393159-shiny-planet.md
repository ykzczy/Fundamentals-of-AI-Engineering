## 更新执行脚本模板

### Python 脚本方法 (推荐)

```python
import json
import re

def update_notebook(filepath, replacements):
    """
    Update markdown cells in a notebook.
    
    Args:
        filepath: Path to .ipynb file
        replacements: List of (old_pattern, new_pattern) tuples
    """
    with open(filepath, 'r', encoding='utf-8') as f:
        notebook = json.load(f)
    
    updated = False
    for cell in notebook['cells']:
        if cell['cell_type'] == 'markdown':
            source = ''.join(cell['source'])
            new_source = source
            
            for old, new in replacements:
                new_source = new_source.replace(old, new)
            
            if new_source != source:
                cell['source'] = [new_source] if len(new_source.split('\n')) == 1 else new_source.split('\n')
                # Handle multi-line strings properly
                if '\n' in new_source:
                    cell['source'] = [line + '\n' for line in new_source.split('\n')[:-1]] + [new_source.split('\n')[-1]]
                updated = True
    
    if updated:
        with open(filepath, 'w', encoding='utf-8') as f:
            json.dump(notebook, f, indent=1, ensure_ascii=False)
        print(f"Updated: {filepath}")
    else:
        print(f"No changes: {filepath}")

# Week 3 examples
update_notebook('week_03/01_local_inference_setup.ipynb', [
    ('# Week 5 — Part 01', '# Week 3 — Part 01'),
    ('Why it matters here (Week 5)', 'Why it matters here (Week 3)'),
])

update_notebook('week_03/04_timeouts_failures.ipynb', [
    ('# Week 4 — Part 01', '# Week 3 — Part 04'),
    ('Why it matters here (Week 4)', 'Why it matters here (Week 3)'),
])

# 更多文件...
```

## 与 Markdown 文件对照去重策略

### 目标
确保 Notebook (.ipynb) 和 Markdown (.md) 文件内容一致，避免重复说明，以 **Markdown 为权威来源**。

### 对照流程

1. **并行打开对应文件**
   - `week_XX/YY_*.md` ←→ `week_XX/YY_*.ipynb`
   - 例如：`01_tokens_context.md` ←→ `01_tokens_context.ipynb`

2. **内容对比检查点**

| 检查项 | Markdown (.md) | Notebook (.ipynb) |
|--------|-----------------|-------------------|
| 标题一致性 | 保留完整标题 | 保留完整标题（需一致） |
| 学习目标 | 保留完整列表 | 简化为指向 Markdown 的链接 |
| 理论解释 | 保留详细内容 | 简化为指向 Markdown 的链接 |
| 代码示例 | 可有代码片段，但必须指向 Notebook 详细实现 | 保留完整可执行代码 |
| 练习任务 | 删除详细步骤，改为指向 Notebook 的链接 | 保留完整交互式练习 |
| Pre-study | 两个文件都保留（导航信息） | 两个文件都保留（导航信息） |

3. **Notebook 文件修改规则**
