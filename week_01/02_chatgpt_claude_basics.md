# Week 1 — Part 02: ChatGPT and Claude Basics

## Overview

ChatGPT and Claude are the most popular browser-based AI assistants. This tutorial helps you get started with both.

## ChatGPT

### What is ChatGPT?

ChatGPT is OpenAI's conversational AI. It's designed for general-purpose tasks:

- Writing and editing text
- Answering questions
- Explaining concepts
- Generating code
- Brainstorming ideas

### Setup checklist

| Step | Action | Success looks like |
|------|--------|-------------------|
| 1. Create account | Go to chatgpt.com, click "Sign Up" or log in | Account verified |
| 2. Choose access level | Free access is sufficient when available | Can send messages |
| 3. First message | Type a prompt and hit Enter | Receive a response |

### Interface basics

| Element | Purpose |
|---------|---------|
| **Chat input** | Type your prompts here |
| **Response area** | AI responses appear here |
| **New chat** | Start fresh conversation |
| **History** | Access previous chats |

### Your first prompts

Try these:

```text
Hello! I'm new to AI tools. What can you help me with?
```

```text
Explain what a "context window" means in AI, 
using an analogy for a non-technical person.
```

```text
Write a short email (under 150 words) thanking 
a colleague for their help on a project.
```

## Claude

### What is Claude?

Claude is Anthropic's AI assistant, known for:

- Handling longer documents
- Detailed explanations
- Nuanced reasoning
- Safety and honesty

### Setup checklist

| Step | Action | Success looks like |
|------|--------|-------------------|
| 1. Create account | Go to claude.ai, click "Sign Up" | Account verified |
| 2. First message | Type a prompt and hit Enter | Receive a response |

| Strength | When to use |
|----------|-------------|
| **Long documents** | Summarizing or analyzing lengthy text |
| **Detailed reasoning** | Complex problems requiring step-by-step analysis |
| **Honest assessment** | When you want candid feedback |

## Effective Prompt Patterns

### Pattern 1: The Explainer

**Template:**
```text
Explain [topic] in simple terms, as if teaching 
someone new to this subject. Use analogies.
```

**Example:**
```text
Explain what an "API" is in simple terms, 
as if teaching someone with no tech background.
Use analogies.
```

### Pattern 2: The Writer

**Template:**
```text
Write a [type] about [topic] that is [length] 
for [audience]. The tone should be [tone].
```

**Example:**
```text
Write a professional email to my manager 
requesting approval for a training workshop. 
Keep it under 200 words. Tone: respectful but confident.
```

### Pattern 3: The Analyzer

**Template:**
```text
Analyze this [content] and provide [analysis type]:
[insert content]

Focus on: [what matters]
```

**Example:**
```text
Analyze this email draft and provide feedback on clarity:

"Hey boss, I need Friday off for a wedding. Thanks."

Focus on: professionalism and clarity.
```

### Pattern 4: The Helper

**Template:**
```text
I'm trying to [goal]. Here's my situation: [context].
Can you help me [specific request]?
```

**Example:**
```text
I'm trying to write a resume for a data analyst role.
My situation: 3 years in marketing, completed data course.
Can you help me draft a summary section?
```

## Common Pitfalls

### Pitfall 1: One giant prompt

**Symptom**: Complex request in one message gives poor results.

**Fix**: Break into steps:
```text
Step 1: "Outline a resume for a data analyst position."
Step 2: "Now help me write the summary section based on this outline."
Step 3: "Review the summary and suggest improvements."
```

### Pitfall 2: No iteration

**Symptom**: First response isn't perfect, but you stop there.

**Fix**: Refine:
- "Make it shorter"
- "Add an example"
- "Change the tone"
- "Explain in simpler terms"

### Pitfall 3: Missing format specification

**Symptom**: Output isn't in the format you wanted.

**Fix**: Specify format:
- "Format as a bullet list"
- "Use paragraphs with headings"
- "Create a table comparing X and Y"

### Pitfall 4: Not saving good prompts

**Symptom**: You forget prompts that worked well.

**Fix**: Keep a prompt library:
- Copy successful prompts to a note
- Organize by task type
- Reuse and refine over time

## Tips for Better Results

1. **Be specific about length**: "Under 200 words" vs "short"
2. **Provide context**: "I'm a beginner" shapes responses
3. **Specify format**: "Bullet list", "table", "paragraphs"
4. **Iterate**: Don't accept first response if it's not right
5. **Verify**: Check facts, don't blindly trust
